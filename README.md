## 📋 Project Description
API testing on Urban.Grocers backend services, validating REST endpoints, HTTP responses, and business rules using Postman.

---

### 📂 Project Resources & Assets
* **📋 Requirements:** [View Functional Requirements (PDF)](https://practicum-content.s3.us-west-1.amazonaws.com/new-markets/qa-sprint-3/QA_3.1.1_Requisitos_para_el_back-end_de_Urban.grocers.pdf)

* **🚀 API Documentation (Endpoints):**
The project interacts with both REST and SOAP services. Below is an example of the **warehouse service validation:**

| Endpoint | Method | Format | Description |
| :--- | :--- | :--- | :--- |
| `/big-world/wsdl` | POST | SOAP (XML) | Comprobación de stock en el almacén "Big World". |
| `/api/v1/users` | POST | JSON | Creación de cuenta de usuario. |

> **Nota:** The full documentation is integrated within the project container. I have included an export of the Postman collection with request examples in the `/docs`.
